### 1. Hana's queue card not showing "In Service" (manicurist ↔ queue desync)
- **Symptom:** Hana (mani-8) was in service with walk-in client Jackie (Gel Pedicure). Turns were correct (`total_turns: 2`), but her queue card showed "available" instead of "In Service."
- **Root cause:** The card's In-Service block (`ManicuristCard.tsx`) is driven only by the `manicurists` row — it renders when `status === 'busy'` and a `currentClient` exists. It does NOT read `queue_entries`. Jackie's queue entry (`1adf5568-c43c-430a-9836-40c8ae34842d`) was `status: inProgress` / `assigned_manicurist_id: mani-8` correctly, but Hana's manicurist row was never flipped to `busy` / `current_client_id`. The turn engine fired but the `status: 'busy'` + `currentClientId` write on the manicurist record was dropped. Same register↔queue sync class of bug seen before.
- **Temp fix applied (live DB, 6/6):**
  ```sql
  update manicurists
  set status = 'busy', current_client_id = '1adf5568-c43c-430a-9836-40c8ae34842d'
  where id = 'mani-8';
  ```
